見積もり詳細ページの比較分析パネル（数量が2つ以上の明細）に追加：

- **💰 金額準拠の理想**（緑カード）— 単価が最も低い数量。顧客にとって最もお得な発注量
- **⚡ 効率準拠の理想**（青カード）— 生産性（部/h）が最も高い数量。工場の稼働効率が最大になる量
- 両方が同じ数量の場合は1枚に統合し「金額・効率ともに同じ数量が最適」と表示
- テーブル行も 💰最安 / ⚡高効率 バッジで色分け（緑 / 青）

## 技術メモ

### デプロイ方法
```
git add app/
git commit -m "feat: ..."
git push origin master
```
→ Vercel が自動でビルド＆デプロイ（1〜2分で反映）

### Tailwind v4 CSS変数問題
`bg-primary` / `bg-popover` 等のCSS変数クラスが機能しない。
→ **全て具体色（`bg-blue-600` 等）で実装すること**

### DB接続
- Neon PostgreSQL (Serverless)
- 環境変数: `DATABASE_URL`（Vercelダッシュボードで管理）

### 認証
- JWT + Cookie (`pq_token`, 8h, HttpOnly)
- Roles: `admin`（全機能）/ `staff`（自分の見積のみ）

### セットアップAPI（初回のみ）
新テーブルが必要な場合はブラウザのコンソールで：
```js
fetch('/api/setup/additional-masters', {method:'POST'}).then(r=>r.json()).then(console.log)
```
