### 🔍 調査で判明した根本原因（価格が間違って取得される問題）

「アマチュアビジランテ8巻」で自動入力すると実売792円のところ800円になる、という報告を起点に調査。判明した事実：

| 情報源 | 状態 |
|---|---|
| 版元ドットコム(hanmoto) API | ❌ `api.hanmoto.com`が完全にダウン（301→404） |
| OpenBD `/v1/search`（タイトル検索） | ❌ エンドポイント自体が存在しない（常に404、以前からの潜在バグ） |
| OpenBD `/v1/get`（ISBN直接取得） | ✅ 正常。ただし該当巻のISBN登録が発売直後で間に合っていないことがある |
| 楽天Books API | ❌→✅ 旧エンドポイント`app.rakuten.co.jp`が仕様変更で廃止されており、`openapi.rakuten.co.jp` + `accessKey`必須の新方式に未対応だった。新規キー取得＋エンドポイント修正で復旧 |
| iTunes | ✅ 稼働中だが**電子書籍版の価格**（紙版と異なることがある） |

→ 紙版データが取れる情報源が軒並み機能しておらず、電子書籍版価格（iTunes）にフォールバックしていたのが誤価格の直接原因。楽天Books API復旧により紙版の正しい価格・出版社が取得できるようになった（実際に792円・講談社で取得確認済み）。

### 技術メモ

#### 楽天Books API（新方式）
```
旧: https://app.rakuten.co.jp/services/api/BooksBook/Search/20170404?applicationId=...
新: https://openapi.rakuten.co.jp/services/api/BooksBook/Search/20170404?applicationId=...&accessKey=...
```
- `accessKey`は`pk_`から始まる形式（新しい楽天ウェブサービスの開発者ポータルで発行）
- Referer/Origin等ブラウザ由来のヘッダー検証があり、`curl`単体では`REQUEST_CONTEXT_BODY_HTTP_REFERRER_MISSING`等で弾かれることがある（ブラウザ実機では問題なし）
- コード内の`RAKUTEN_APP_ID_DEFAULT`/`RAKUTEN_ACCESS_KEY_DEFAULT`はフォールバック用。実際に使われる値は`getRakutenAppId()`/`getRakutenAccessKey()`経由でFirebase設定（`userSettings.rakutenAppId`等）があればそちらを優先

#### ローカル動作確認用に`launch.json`を追加
`dejiina_agent/.claude/launch.json` に `manga-local`（`npx serve .. -p 8766`）を追加。Preview系ツールで manga.html を直接検証できるようにした。
